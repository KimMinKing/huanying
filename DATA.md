## 2. 메신저 상담 채널

> 중국인 고객 유치의 핵심 채널. 특히 WeChat은 차별화 포인트.

**현재 상태**: 핸들만 적혀 있고 URL/QR은 비어 있음 (`siteConfig.kakao`, `siteConfig.wechat`, 그리고 `contactChannels` in `src/data/chineseServices.ts`)

### KakaoTalk 채널

**발급 방법**:
1. https://biz.kakao.com/ 접속 → 사업자 정보로 로그인
2. '채널' 만들기 → 채널명, 프로필 사진, 검색 ID 설정
3. 채널 URL 발급: `https://pf.kakao.com/_xxxxxx` 형태

**입력位置**: `src/data/comparison.ts` → `siteConfig.kakao`

| 필드 | 현재 | 실제 값 |
|---|---|---|
| 검색 ID (`handle`) | `@라이플` | ___________________ |
| 채널 URL (`url`) | (빈 문자열) | `https://pf.kakao.com/_______` |

### WeChat

**발급 방법**:
1. WeChat 계정 생성 (본인/담당자 중국어 가능자)
2. WeChat ID 설정 (영문+숫자, 6~20자)
3. QR 코드 생성: WeChat → Me → Profile → "My QR Card" → 저장

**입력位置**: `src/data/comparison.ts` → `siteConfig.wechat`, `src/data/chineseServices.ts` → `contactChannels.wechat`

| 필드 | 현재 | 실제 값 |
|---|---|---|
| WeChat ID (`handle`) | `lifful_kr` | ___________________ |
| QR 이미지 URL (`qrUrl`) | (빈 문자열) | `https://your-cdn.com/wechat-qr.png` |

**QR 이미지 업로드**:
- 정사각형 PNG/SVG, 300×300 이상 권장
- CDN(AWS S3, Cloudinary) 또는 `public/wechat-qr.png` 경로에 직접 업로드

### Line (선택)

일본/대만 고객 대상 확장 시 추가. 현재는 미사용.

## 3. 관리자 계정

> 초기 admin 계정 정보. **운영 서버 배포 전 반드시 비밀번호 변경.**

**현재 상태**: 스키마에 시드로 admin 1행 삽입 (`db/schema.sql`)

| 항목 | 현재 |
|---|---|
| 이메일 | `admin@lifful.com` |
| 비밀번호 | `admin123` |
| 역할 | `admin` |

**비밀번호 변경 방법**:

```bash
# 1) 새 비밀번호의 bcrypt 해시 생성
node -e "console.log(require('bcryptjs').hashSync('새비밀번호', 10))"

# 2) DB 업데이트
docker exec -it lifful-postgres psql -U lifful -d lifful -c \
  "UPDATE staff SET password_hash='여기에_해시' WHERE email='admin@lifful.com';"
```

**새 관리자 추가 방법**:

```sql
INSERT INTO staff (email, name, password_hash, role, language)
VALUES (
  'newadmin@lifful.com',
  '새 관리자 이름',
  'bcrypt 해시값',
  'admin',
  'ko'
);
```
